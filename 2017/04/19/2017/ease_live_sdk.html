环信直播SDK使用

  • ease
  • live
  • sdk
android

开发直播的话，有很多的平台可供选择，因为之前即时通讯的软件用的是环信的sdk，看到他也提供了直播的sdk，拿来学习一下。
环信的直播是基于自己的聊天室和Ucloud的直播服务，来实现的。

注册并创建应用

这一步可以直接看环信的官方教程啦，看这里查看

下载sdk

直接下载官网提供的sdk连接

下载成功后，先修改build.gradle等编译的配置，使用自己Android Studio已经存在的就好，然后导入到Android Studio中。

用户体系介绍

在开发之前，需要先了解一下用户体系，不同用户的token是不一样的，对应的权限也是不一样的，有的操作只有特定的用户才能操作
官方的介绍在这里

需要特别说明的是获取管理员的Token

获取 APP 管理员 Token

环信提供的 REST API 需要权限才能访问，权限通过发送 HTTP 请求时携带 token 来体现，下面描述获取 token 的方式。说明：API 描述的时候使用到的 {APP 的 client_id} 之类的这种参数需要替换成具体的值。

重要提醒：获取 token 时服务器会返回 token 有效期，具体值参考接口返回的 expires_in 字段值。由于网络延迟等原因，系统不保证 token 在此值表示的有效期内绝对有效，如果发现 token 使用异常请重新获取新的 token，比如 http response code 返回 401。另外，请不要频繁向服务器发送获取 token 的请求，同一账号发送此请求超过一定频率会被服务器封号，切记，切记！！

使用 APP 的 client_id 和 client_secret 获取授权管理员 token
注意：在调用程序中，如果返回429或503错误，说明接口被限流了，请稍微暂停一下并重试。详见接口限流说明。

client_id 和 client_secret 可以在环信管理后台的 APP 详情页面看到。

Key Value
access_token token 值
expires_in token 有效时间，以秒为单位，在有效期内不需要重复获取
application 当前 APP 的 UUID 值

可能的错误码：400（client_id 或 client_secret 错误）、5xx。详见：服务器端REST API常见错误码

创建主播

需要说明的是，直接编译demo生成的应用，注册账号后，只是普通的用户，可以观看直播，但是不能创建直播的。
新建主播接口，只支持REST API，SDK应该没有提供。这个接口应该是在console后台上操作，调用的。
(另外：SDK上不要用任何app admin 的token，这样很危险)
创建直播需要主播的权限，可以参考一下API来操作。

新建主播接口1
调用来源

console

权限要求

app管理员

字段说明
  • username 用户的环信ID;
  • liveroom_id 固定field，后跟默认该主播关联的直播聊天室ID

必须要求传入一个直播聊天室ID。

设置直播推流，拉流的基础地址

用户在使用直播聊天室之前，首先要调用环信的REST API ，来设置直播推流，拉流的基础地址； 每个直播房间的推流，拉流地址，都是以此作为基础地址的。

设置(更新)appkey的 直播流地址(pc pull/pc push/mobile pull/mobile push)
调用来源

REST API（console后台）

权限要求

appadmin

字段说明
  • pc_pull PC端拉流地址
  • pc_pushPC端推流地址
  • mobile_pull MOBILE端拉流地址
  • mobile_push MOBILE端推流地址
