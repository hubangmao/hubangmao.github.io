使用Volley上传头像图片

  • java
  • post
  • volley
android

在Gooogle I/O 2013年的大会上面，展示的Volley库，已经成为android开发中最常用的处理和缓存网络请求的库。
如果你还没有看过Gooogle　I/O中关于Volley的介绍,在继续这篇文章之前我建议你先去看看关于Volley的介绍，对它有一些基本的理解。
中文的话可以看下这篇翻译的文章。
关于Volley的使用网上有很多的教程，这里主要讲一下如何使用Volley上传图片的问题。
这里上传的图片是头像，所以图片的大小已经固定好了。
我们有两种方式可以完成上传图片到服务器：

  • 使用Multipart Request
  • 发送图片的字符串到服务器
    我们使用Volley都可以实现。

    使用Multipart Request的方式

    Multipart Request是一种HTTP请求，可以通过HTTP客户端或者浏览器发送文件和数据到HTTP服务器，Volley 支持这种类型。

主要思路：
将需要上传的图片转换为MultipartRequest请求内容需要的数组格式，通过MultipartRequest上传到服务器，在服务器端解析。

1. 首先将我们需要上传的图片转换为数组数据。
2. 调用MultipartRequest请求
3. 自定义回调函数，可以添加对服务器请求结果的处理
4. 自定义MultipartRequest，需要继承Request，并且重写getBodyContentType方法和getBody方法，分别返回传过来的内容和内容类型。
5. 服务器端解析数据并处理

使用发送图片字符串的方式

我们这里使用的图片字符串为Base64编码的字符串。

Base64是网络上最常见的用于传输8Bit字节代码的编码方式之一，大家可以查看RFC2045～RFC2049，上面有MIME的详细规范。Base64编码可用于在HTTP环境下传递较长的标识信息。例如，在Java Persistence系统Hibernate中，就采用了Base64来将一个较长的唯一标识符（一般为128-bit的UUID）编码为一个字符串，用作HTTP表单和HTTP GET URL中的参数。在其他应用程序中，也常常需要把二进制数据编码为适合放在URL（包括隐藏表单域）中的形式。此时，采用Base64编码具有不可读性，即所编码的数据不会被人用肉眼所直接看到。

主要思路：
将需要上传的图片转换为Base64编码的字符串，通过Volley的StringRequest请求发生给服务器，并在服务器端解码。

1. 根据图片存储的地址，将其转化为Base64编码字符串
2. 使用Volley的StringRequest请求，并重写getParams方法，将图片的字符串以参数的形式传递。
3. 自定义回调函数，可以添加对服务器请求结果的处理
4. 服务器端解析数据并处理

好啦，以上就是使用Volley上传图片的两种方法，希望能帮助到大家。
